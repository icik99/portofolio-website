The *Duta Anak Indonesia* project is a professional web solution designed to empower and engage children across Indonesia. It serves as a platform for the selection of child ambassadors representing each province, fostering opportunities and collaboration for the younger generation.  

As the **Frontend Developer**, I was responsible for creating two key components of this project:  

1. **The Main Portal Website**  
2. **The CMS Portal for Administrators**  

![Main Portal Screenshot](/duta-anak/1.png)  
![Main Portal Screenshot](/duta-anak/2.png)  

## Main Portal Website  

The main portal acts as the public-facing platform, providing comprehensive information, resources, and opportunities for children and stakeholders. Built with:  

- **TypeScript** for type-safe development  
- **Next.js** for its server-side rendering and optimized performance  
- **Tailwind CSS** for responsive and scalable styling  

I focused on implementing a visually appealing, user-friendly interface, ensuring accessibility standards, and optimizing the overall user experience.  

## CMS Portal  

The CMS portal was designed to empower administrators with seamless content management capabilities. Key features include:  

- **Dynamic Content Updates**: Simplified updates for website administrators.  
- **Streamlined Workflows**: Intuitive design to enhance admin efficiency.  
- **Consistent Design Language**: Maintaining the look and feel of the main portal.  

This component ensures that the platform remains dynamic and up-to-date with minimal overhead for the management team.  

## Project Technologies  

- **Programming Language**: TypeScript  
- **Framework**: Next.js  
- **Styling**: Tailwind CSS  
- **Tools**: GitHub, Jira, Postman, Docker 

## Collaboration and Tools  

The success of this project was supported by effective collaboration with the team and leveraging key tools and methodologies:  

- **GitHub**: For version control and collaborative development.  
- **Jira**: Used for project management with the _**Agile Scrum methodology**_, ensuring iterative progress and clear task allocation.  
- **Postman**: Utilized to test and validate APIs provided by the backend team.  
- **Docker**: Implemented for containerization and deployment, ensuring consistent environments across development, staging, and production.
